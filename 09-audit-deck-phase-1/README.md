================================================================================
README — AuditDeck Phase 1
Optiv Managed Security Services
================================================================================

BEFORE YOU START
--------------------------------------------------------------------------------

Before working through this package please make sure you have completed the
following from your ServiceNow ticket:

    Task 1 — Reviewed the MSS - Microsoft Sentinel Audit workbook in your
    Sentinel environment and confirmed the table list looks as expected.

    Task 2 — Confirmed the status of your Microsoft Defender XDR Data Connector
    and replied in the ticket. If you need guidance on this please open
    05-DefenderXDR-SetupGuide.docx before proceeding.

If you have not completed Tasks 1 and 2 please do those first and reply in
your ticket before continuing here.

--------------------------------------------------------------------------------
WHAT WE NEED FROM YOU
--------------------------------------------------------------------------------

By the end of this process we need three things confirmed and delivered:

    1. The AuditDeck workbook tables have been reviewed and acknowledged

    2. The Microsoft Defender XDR Data Connector status is confirmed and
       understood by both parties — either already configured and verified
       or a separate ticket has been opened to address it

    3. The following two completed files returned to us via secure Kiteworks link:

       02-ClientInformation.xlsx — Tab 1 filled in with your information
       04-EnvironmentProfile-Template.docx — filled in describing your environment

--------------------------------------------------------------------------------
WHAT IS IN THIS PACKAGE
--------------------------------------------------------------------------------

    README.docx (this file)
    Start here.

    01-ClientInformation-Guide.docx
    A field by field guide explaining the information we are asking for
    in the intake spreadsheet.

    02-ClientInformation.xlsx
    The intake spreadsheet. Tab 1 is your template to fill in. Tab 2 is
    a completed example using a fictional company for reference.

    03-EnvironmentProfile-Guide.docx
    A guide that walks through each section of the environment profile
    document and what we are looking for.

    04-EnvironmentProfile-Template.docx
    The environment profile document for you to fill in.

    05-DefenderXDR-SetupGuide.docx
    A guide explaining the Microsoft Defender XDR Data Connector — what
    it is, why it matters, and how to check if it is configured in your
    environment.

--------------------------------------------------------------------------------
COMPLETING THE INTAKE FORMS
--------------------------------------------------------------------------------

    02-ClientInformation.xlsx
    Open this file and fill in the Value column on Tab 1 only. Tab 2 is
    a completed example for reference — do not edit it. Read
    01-ClientInformation-Guide.docx alongside this file for detailed
    guidance on every field.

    04-EnvironmentProfile-Template.docx
    Fill in each section describing your technology environment. Read
    03-EnvironmentProfile-Guide.docx alongside this file for guidance
    on what we are looking for in each section. Do your best and leave
    anything blank that does not apply or that you are unsure about.

--------------------------------------------------------------------------------
RETURNING YOUR COMPLETED FILES
--------------------------------------------------------------------------------

When you have completed both documents reply in your ServiceNow ticket with
a contact email address. We will send you a secure Kiteworks link to upload
the following two files:

    Completed 02-ClientInformation.xlsx
    Completed 04-EnvironmentProfile-Template.docx

Please do not send files via email.

--------------------------------------------------------------------------------
QUESTIONS
--------------------------------------------------------------------------------

If you have any questions at any point please reply directly in your
ServiceNow ticket and we will be happy to help.
